### <a name="target-view"></a>Целевое представление

***Целевое представление** _ — это область конструирования и макета экрана WYSIWYG. Это представление предназначено для представления физического экрана или экранов, доступных на целевом оборудовании. Объекты можно выбирать, перемещать, изменять их размер и т. д. с помощью простых операций мыши. Кроме того, для выбранных объектов в целевом представлении доступны операции выравнивания и порядка наложения в виде кнопок. Выбор объекта в _*_целевом представлении_*_ также приведет к отображению свойств этого объекта в _*_представлении свойств_*_. На _*_рис. 6_*_ ниже показано _*_целевое представление_** GUIX Studio.

## <a name="the-guix-studio-project"></a>Проект GUIX Studio

Проект GUIX Studio содержит сведения о дизайне экрана пользовательского интерфейса и его ресурсах. Данные проекта сохраняются в XML-файле с расширением .***gxp***. Поскольку файл проекта представляет собой схему XML, ему можно присваивать версии и, таким образом, им можно управлять и его можно совместно использовать так же, как любой другой исходный файл.

При первом запуске в GUIX Studio необходимо открыть один из образцов проекта, имеющихся в дистрибутиве, или создать новый проект. Вся работа сохраняется в файле данных проекта.

GUIX Studio также создает исходные файлы ANSI C. Эти исходные файлы содержат ресурсы приложения или структуры данных, описывающие разрабатываемые экраны. GUIX Studio также записывает в эти созданные исходные файлы функции API, которые используют созданные структуры данных для динамического создания экранов приложения. Программное обеспечение приложения будет просто вызывать имеющиеся функции API для создания экранов, разработанных в GUIX Studio.

В ходе разработки пользовательского интерфейса инструмент GUIX Studio можно использовать для периодического создания выходных файлов, совместимых с GUIX, с помощью которых можно собирать и запускать разработанный интерфейс. Вы можете компилировать и запускать созданные исходные файлы на целевом оборудовании или на настольном ПК с ОС Windows, который имитирует ThreadX и GUIX.

## <a name="guix-studio-project-organization"></a>Организация проекта в GUIX Studio

Некоторые знания о базовой организации проекта в GUIX Studio помогут понять, как можно эффективно использовать GUIX Studio, и применять информацию, имеющуюся в представлении проекта в интегрированной среде разработки GUIX Studio. Представление проекта представляет собой сводное визуальное представление всей информации, содержащейся в проекте.

Переходя к описанию проекта, необходимо дать несколько определений. Во-первых, мы используем термин «**дисплей**» для обозначения физического устройства отображения. Чаще всего это ЖК-дисплей, но он может работать на основе других технологий. Следующий термин — «**экран**», означающий объект GUIX верхнего уровня, обычно это окно GUIX и все связанные с ним дочерние элементы. Экран — это программная конструкция, которую можно определить и изменить во время выполнения. Наконец, «**тема**» — это коллекция ресурсов. Тема включает таблицу определений цветов, определений шрифтов и определений карт отображения пикселей, предназначенных для совместной работы и предоставления конечному пользователю единообразного внешнего вида и функционала.

Во-первых, проект включает набор глобальных данных, таких как имя проекта, число поддерживаемых дисплеев, разрешение и формат цвета каждого дисплея, число поддерживаемых языков, имя каждого поддерживаемого языка. Имя проекта — это первый узел, отображаемый в представлении проекта.

Далее в проекте в упорядоченном виде содержится вся информация, необходимая для 4 физических дисплеев (макс.), а также экранов и ресурсов, доступных для каждого экрана. Отображаемые имена — это узлы следующего уровня в дереве представления проекта.

Уникальная функция приложения GUIX Studio — это встроенная поддержка нескольких физических дисплеев, каждый из которых имеет собственное разрешение (x, y), цветовой формат, экраны и ресурсы. Хотя подавляющее большинство приложений GUIX предназначены только для одного физического дисплея, эта возможность важна для тех, кто делает продукт, который должен поддерживать несколько одновременно работающих физических дисплеев.

Под каждым определением дисплея расположены окна верхнего уровня и экраны, заданные для этого дисплея. Определения экранов могут быть вложены на любом уровне в зависимости от наличия вложенных дочерних мини-приложений и их числа на каждом экране.

Эта организация экранов и дочерних мини-приложений отображается в графическом виде в представлении проекта.

Кроме того, с каждым дисплеем связаны темы, поддерживаемые дисплеем, и содержимое ресурсов, из которых состоит тема. Если проект содержит несколько дисплеев, вы заметите, что содержимое представления ресурсов меняется в зависимости от выбранного дисплея. Это связано с тем, что содержимое ресурсов связано с конкретным дисплеем. Для каждого физического дисплея может отличаться не только формат цвета, но и карты отображения пикселей, цвета и шрифты, доступные для использования.

Последним компонентом, поддерживаемым проектом, являются данные таблицы строк, связанные с каждым из дисплеев. Так как дисплеи могут иметь совершенно разные разрешения (x, y), строковые данные сохраняются отдельно для каждого дисплея, заданного в проекте.
